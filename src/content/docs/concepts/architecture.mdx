## 系統架構

OpenClaw 採用模組化架構設計，主要由以下元件組成：

```
┌──────────────────────────────────────────┐
│              聊天頻道層                    │
│   LINE │ Telegram │ WhatsApp │ Discord   │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│              閘道器 (Gateway)              │
│   訊息路由 │ 身份驗證 │ 速率限制          │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│              代理引擎 (Agent Engine)       │
│   對話管理 │ 記憶 │ 外掛系統              │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│              模型層 (Model Layer)          │
│   OpenAI │ Anthropic │ Google │ 本地模型  │
└──────────────────────────────────────────┘
```

## 核心元件

### 閘道器 (Gateway)

閘道器是 OpenClaw 的入口點，負責接收來自各種聊天頻道的訊息，並將回應發送回去。它處理：

- 訊息格式轉換
- 身份驗證與授權
- 速率限制與防濫用
- 多頻道路由

### 代理引擎 (Agent Engine)

代理引擎是 OpenClaw 的核心，負責處理對話邏輯：

- 上下文管理
- 記憶系統
- 外掛執行
- 工具呼叫

### 模型層 (Model Layer)

模型層提供統一的介面來存取不同的 AI 模型提供者，支援熱切換和負載均衡。

## 資料流程

1. 使用者透過聊天 App 發送訊息
2. 閘道器接收並驗證訊息
3. 代理引擎載入使用者的上下文和記憶
4. 組合提示詞（prompt）發送給 AI 模型
5. 接收模型回應並執行後處理
6. 透過閘道器將回應發送回聊天 App
