本頁面涵蓋 **LLM/模型供應商**（非 WhatsApp/Telegram 等聊天頻道）。
有關模型選擇規則，請參閱 [模型](/docs/concepts/models)。

## 快速規則

- 模型參照使用 `provider/model` 格式（例如：`opencode/claude-opus-4-6`）。
- 若設定了 `agents.defaults.models`，它會成為允許清單。
- CLI 輔助指令：`openclaw onboard`、`openclaw models list`、`openclaw models set <provider/model>`。

## API 金鑰輪替

- 支援特定供應商的通用金鑰輪替。
- 透過以下方式設定多組金鑰：
  - `OPENCLAW_LIVE_<PROVIDER>_KEY`（單一即時覆寫，最高優先順序）
  - `<PROVIDER>_API_KEYS`（逗號或分號分隔的清單）
  - `<PROVIDER>_API_KEY`（主要金鑰）
  - `<PROVIDER>_API_KEY_*`（編號清單，例如 `<PROVIDER>_API_KEY_1`）
- 對於 Google 供應商，`GOOGLE_API_KEY` 也作為後備。
- 金鑰選擇順序會保持優先順序並去除重複值。
- 請求僅在速率限制回應（例如 `429`、`rate_limit`、`quota`、`resource exhausted`）時使用下一個金鑰重試。
- 非速率限制的失敗會立即失敗，不會嘗試金鑰輪替。
- 當所有候選金鑰都失敗時，會返回最後一次嘗試的最終錯誤。

## 內建供應商（pi-ai 目錄）

OpenClaw 隨附 pi-ai 目錄。這些供應商**無需** `models.providers` 設定；只需設定驗證方式並選擇模型即可。

### OpenAI

- Provider：`openai`
- Auth：`OPENAI_API_KEY`
- 選用輪替：`OPENAI_API_KEYS`、`OPENAI_API_KEY_1`、`OPENAI_API_KEY_2`，加上 `OPENCLAW_LIVE_OPENAI_KEY`（單一覆寫）
- 範例模型：`openai/gpt-5.1-codex`
- CLI：`openclaw onboard --auth-choice openai-api-key`

```json5
{
  agents: { defaults: { model: { primary: "openai/gpt-5.1-codex" } } },
}
```

### Anthropic

- Provider：`anthropic`
- Auth：`ANTHROPIC_API_KEY` 或 `claude setup-token`
- 選用輪替：`ANTHROPIC_API_KEYS`、`ANTHROPIC_API_KEY_1`、`ANTHROPIC_API_KEY_2`，加上 `OPENCLAW_LIVE_ANTHROPIC_KEY`（單一覆寫）
- 範例模型：`anthropic/claude-opus-4-6`
- CLI：`openclaw onboard --auth-choice token`（貼上 setup-token）或 `openclaw models auth paste-token --provider anthropic`

```json5
{
  agents: { defaults: { model: { primary: "anthropic/claude-opus-4-6" } } },
}
```

### OpenAI Code (Codex)

- Provider：`openai-codex`
- Auth：OAuth (ChatGPT)
- 範例模型：`openai-codex/gpt-5.3-codex`
- CLI：`openclaw onboard --auth-choice openai-codex` 或 `openclaw models auth login --provider openai-codex`
- 預設傳輸方式為 `auto`（優先使用 WebSocket，SSE 作為後備）
- 可透過 `agents.defaults.models["openai-codex/<model>"].params.transport` 覆寫每個模型的傳輸方式（`"sse"`、`"websocket"` 或 `"auto"`）

```json5
{
  agents: { defaults: { model: { primary: "openai-codex/gpt-5.3-codex" } } },
}
```

### Google Gemini（API 金鑰）

- Provider：`google`
- Auth：`GEMINI_API_KEY`
- 選用輪替：`GEMINI_API_KEYS`、`GEMINI_API_KEY_1`、`GEMINI_API_KEY_2`、`GOOGLE_API_KEY` 後備，以及 `OPENCLAW_LIVE_GEMINI_KEY`（單一覆寫）
- 範例模型：`google/gemini-3-pro-preview`
- CLI：`openclaw onboard --auth-choice gemini-api-key`

### Google Vertex、Antigravity 及 Gemini CLI

- Provider：`google-vertex`、`google-antigravity`、`google-gemini-cli`
- Auth：Vertex 使用 gcloud ADC；Antigravity/Gemini CLI 使用各自的驗證流程

> **注意：** OpenClaw 中的 Antigravity 和 Gemini CLI OAuth 為非官方整合。部分使用者回報在使用第三方客戶端後 Google 帳號受到限制。請檢閱 Google 條款，並在選擇使用時使用非關鍵帳號。

- Antigravity OAuth 作為內建外掛（`google-antigravity-auth`，預設停用）。
  - 啟用：`openclaw plugins enable google-antigravity-auth`
  - 登入：`openclaw models auth login --provider google-antigravity --set-default`
- Gemini CLI OAuth 作為內建外掛（`google-gemini-cli-auth`，預設停用）。
  - 啟用：`openclaw plugins enable google-gemini-cli-auth`
  - 登入：`openclaw models auth login --provider google-gemini-cli --set-default`
  - 注意：您**不需**將 client id 或 secret 貼入 `openclaw.json`。CLI 登入流程會將 token 儲存在 Gateway 主機上的 auth profiles 中。

## 透過 `models.providers` 設定的供應商（自訂/base URL）

使用 `models.providers`（或 `models.json`）來新增**自訂**供應商或 OpenAI/Anthropic 相容的代理。

### Moonshot AI (Kimi)

Moonshot 使用 OpenAI 相容端點，因此將其設定為自訂供應商：

- Provider：`moonshot`
- Auth：`MOONSHOT_API_KEY`
- 範例模型：`moonshot/kimi-k2.5`

```json5
{
  agents: {
    defaults: { model: { primary: "moonshot/kimi-k2.5" } },
  },
  models: {
    mode: "merge",
    providers: {
      moonshot: {
        baseUrl: "https://api.moonshot.ai/v1",
        apiKey: "${MOONSHOT_API_KEY}",
        api: "openai-completions",
        models: [{ id: "kimi-k2.5", name: "Kimi K2.5" }],
      },
    },
  },
}
```

### Kimi Coding

Kimi Coding 使用 Moonshot AI 的 Anthropic 相容端點：

- Provider：`kimi-coding`
- Auth：`KIMI_API_KEY`
- 範例模型：`kimi-coding/k2p5`

```json5
{
  env: { KIMI_API_KEY: "sk-..." },
  agents: {
    defaults: { model: { primary: "kimi-coding/k2p5" } },
  },
}
```

模型參照：

- `qwen-portal/coder-model`

### Synthetic

Synthetic 在 `synthetic` 供應商下提供 Anthropic 相容模型：

- Provider：`synthetic`
- Auth：`SYNTHETIC_API_KEY`
- 範例模型：`synthetic/hf:MiniMaxAI/MiniMax-M2.1`
- CLI：`openclaw onboard --auth-choice synthetic-api-key`

```json5
{
  agents: {
    defaults: { model: { primary: "synthetic/hf:MiniMaxAI/MiniMax-M2.1" } },
  },
  models: {
    mode: "merge",
    providers: {
      synthetic: {
        baseUrl: "https://api.synthetic.new/anthropic",
        apiKey: "${SYNTHETIC_API_KEY}",
        api: "anthropic-messages",
        models: [{ id: "hf:MiniMaxAI/MiniMax-M2.1", name: "MiniMax M2.1" }],
      },
    },
  },
}
```

### MiniMax

MiniMax 透過 `models.providers` 設定，因其使用自訂端點：

- MiniMax（Anthropic 相容）：`--auth-choice minimax-api`
- Auth：`MINIMAX_API_KEY`

### Ollama

Ollama 是一個提供 OpenAI 相容 API 的本地 LLM 執行環境：

- Provider：`ollama`
- Auth：無需（本地伺服器）
- 範例模型：`ollama/llama3.3`
- 安裝：[https://ollama.ai](https://ollama.ai)

```bash
# 安裝 Ollama 後，拉取模型：
ollama pull llama3.3
```

```json5
{
  agents: {
    defaults: { model: { primary: "ollama/llama3.3" } },
  },
}
```

Ollama 在本地 `http://127.0.0.1:11434/v1` 執行時會自動偵測。

### vLLM

vLLM 是一個本地（或自架）的 OpenAI 相容伺服器：

- Provider：`vllm`
- Auth：選用（取決於您的伺服器）
- 預設 base URL：`http://127.0.0.1:8000/v1`

若要在本地啟用自動偵測（若您的伺服器不強制驗證，任何值皆可）：

```bash
export VLLM_API_KEY="vllm-local"
```

然後設定模型（替換為 `/v1/models` 回傳的 ID）：

```json5
{
  agents: {
    defaults: { model: { primary: "vllm/your-model-id" } },
  },
}
```

### 本地代理（LM Studio、vLLM、LiteLLM 等）

範例（OpenAI 相容）：

```json5
{
  agents: {
    defaults: {
      model: { primary: "lmstudio/minimax-m2.1-gs32" },
      models: { "lmstudio/minimax-m2.1-gs32": { alias: "Minimax" } },
    },
  },
  models: {
    providers: {
      lmstudio: {
        baseUrl: "http://localhost:1234/v1",
        apiKey: "LMSTUDIO_KEY",
        api: "openai-completions",
        models: [
          {
            id: "minimax-m2.1-gs32",
            name: "MiniMax M2.1",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 200000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

> **提示：** 對於自訂供應商，`reasoning`、`input`、`cost`、`contextWindow` 和 `maxTokens` 為選用。省略時，OpenClaw 預設為：`reasoning: false`、`input: ["text"]`、`cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 }`、`contextWindow: 200000`、`maxTokens: 8192`。建議設定與您的代理/模型限制相符的明確值。
