## 安全優先設計

OpenClaw 從設計之初就將安全和隱私作為核心原則。

## 安全架構

### 本機優先

所有資料預設儲存在你的本機裝置上：

- 對話記錄
- 記憶資料
- 設定和金鑰
- 外掛資料

### 加密儲存

敏感資料使用 AES-256 加密：

```yaml
security:
  encryption:
    enabled: true
    algorithm: aes-256-gcm
```

### 存取控制

控制 AI 代理可以存取的資源：

```yaml
security:
  permissions:
    file_system: read_only
    network: restricted
    shell: disabled
```

## API 金鑰安全

OpenClaw 提供安全的金鑰管理：

```bash
# 使用安全儲存設定金鑰
openclaw secret set openai_api_key

# 列出已儲存的金鑰（不顯示值）
openclaw secret list
```

## 網路安全

如果啟用遠端存取，OpenClaw 提供多層安全保護：

- TLS/SSL 加密傳輸
- 身份驗證 Token
- IP 白名單
- 速率限制

## 安全最佳實踐

1. 定期更新 OpenClaw 至最新版本
2. 使用強密碼保護遠端存取
3. 限制 AI 代理的檔案系統存取權限
4. 定期審查代理的工具使用記錄
5. 不要在公開網路上暴露未加密的 API
