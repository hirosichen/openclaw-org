## 常見問題

### 閘道器無法啟動

**症狀**：執行 `openclaw start` 時出現錯誤。

**解決方案**：

1. 檢查連接埠是否被佔用：

```bash
lsof -i :3000
```

2. 嘗試使用不同的連接埠：

```bash
openclaw start --port 8080
```

3. 檢查設定檔語法：

```bash
openclaw config validate
```

### Webhook 收不到訊息

**症狀**：頻道已設定但收不到訊息。

**解決方案**：

1. 確認 Webhook URL 正確且可從外部存取
2. 檢查 SSL 憑證是否有效
3. 查看閘道器日誌：

```bash
openclaw logs --follow
```

4. 測試 Webhook 端點：

```bash
curl -X POST http://localhost:3000/webhook/line \
  -H "Content-Type: application/json" \
  -d '{"test": true}'
```

### 回應延遲過長

**症狀**：AI 回應時間超過預期。

**解決方案**：

1. 檢查 AI 模型的 API 回應時間
2. 調整超時設定：

```yaml
gateway:
  timeout: 30000  # 毫秒
```

3. 考慮使用更快的模型或本地模型

### 連線被拒

**症狀**：出現 `Connection refused` 錯誤。

**解決方案**：

1. 確認閘道器正在運行
2. 檢查防火牆設定
3. 確認綁定地址是否正確（`0.0.0.0` vs `127.0.0.1`）

## 診斷工具

```bash
# 檢查系統狀態
openclaw status

# 查看詳細日誌
openclaw logs --level debug

# 檢查頻道連線
openclaw channel status
```
