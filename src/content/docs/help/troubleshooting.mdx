## 前 60 秒

依序執行以下診斷指令以快速評估系統狀態：

```bash
openclaw status
openclaw status --all
openclaw gateway probe
openclaw gateway status
openclaw doctor
openclaw channels status --probe
openclaw logs --follow
```

每個指令都有對應的正常輸出指標，包括執行狀態檢查、Gateway 可達性驗證、頻道連線確認和日誌活動監控。

## 決策樹

根據以下八個故障類別選擇對應的診斷路徑：

1. 無回覆
2. 儀表板或 Control UI 連線失敗
3. Gateway 啟動問題
4. 頻道已連線但訊息未流通
5. Cron 或 heartbeat 失敗
6. Node 已配對但工具執行失敗
7. 瀏覽器工具故障

## 無回覆

包含診斷指令和預期輸出模式。常見日誌特徵包括 mention 閘控、待處理配對請求和過濾。深入參考 Gateway 疑難排解、頻道和配對程序文件。

## 儀表板/Control UI

診斷重點在 Gateway 狀態驗證、RPC probe 檢查和驗證確認。常見問題包括 device identity 需求、授權不匹配和不正確的 Gateway URL。

## Gateway 服務

處理啟動和執行問題。常見日誌特徵涉及 Gateway 模式設定、驗證繫結需求和連接埠衝突。參考 Gateway 設定和背景程序文件。

## 頻道訊息流

檢查已連線但無法運作的頻道。診斷重點包括傳輸狀態、配對驗證和 mention 偵測。常見問題涉及 mention 需求、待核准和權限/scope 問題。

## Cron/Heartbeat

```bash
openclaw cron status
```

正常輸出指標包括已啟用的排程器狀態和最近成功的工作執行。常見問題包括已停用的排程器、安靜時段跳過和不存在的帳號目標。

## Node 工具執行

涵蓋已配對但執行失敗的 Node。診斷包括 Node 狀態和能力驗證。常見特徵涉及背景可用性、OS 權限、核准需求和允許清單限制。

## 瀏覽器工具

處理瀏覽器狀態驗證和 profile 確認。常見問題包括 Chrome 啟動失敗、不正確的二進位路徑、擴充功能連線問題和無法連線的 attach-only profile。
