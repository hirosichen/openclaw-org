## 安裝與執行環境

### 系統需求

- 需要 Node.js >= 22；建議使用 pnpm
- 可在 Raspberry Pi 上執行（個人使用最低 512MB-1GB RAM）
- 可透過 npm 或 git-based hackable install 安裝
- 入門設定通常需要 5-15 分鐘

### 驗證

- Claude 訂閱可透過 setup-token（非 API key）運作
- OpenAI Codex 支援 OAuth
- 本機模型可作為 API key 的替代方案

## 設定要點

### 資料儲存

- 狀態存放在 `~/.openclaw`（設定、憑證、工作階段）
- Agent 工作空間可設定，預設為 `~/.openclaw/workspace`
- 記憶持久化在 markdown 檔案中（MEMORY.md、每日筆記）

### Gateway 設定

- 預設連接埠：18789
- 預設啟用 token 驗證（即使在 localhost 上）
- 非 loopback 繫結需要設定驗證
- 熱重載支援大多數設定變更

## 操作指南

### 模型選擇

對於高風險工作使用能負擔的最佳模型，日常聊天則使用較便宜的模型。

### 多 Agent 路由

- 單一 Gateway 可託管多個 agent
- 每個 agent 維護獨立的工作空間和工作階段歷史
- 可實現每個 agent 的專業化工作流程

### 工作階段管理

- 工作階段在可設定的閒置時間後過期（預設 60 分鐘）
- `/new` 指令開始全新對話
- Sub-agent 可用於平行任務執行

## 部署選項

- 筆電/桌機適合主動使用
- VPS 適合全天候可靠性
- Mac mini 作為常駐主機，搭配 node 周邊裝置
- Windows 使用者建議使用 WSL2

OpenClaw 不只是一個 Claude 包裝器，而是一個本機優先的控制平面，支援多個訊息平台、有狀態工作階段和工具整合。
