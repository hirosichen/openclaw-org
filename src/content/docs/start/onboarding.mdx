本指南說明 OpenClaw macOS 應用程式目前的首次執行引導流程。使用者選擇 Gateway 執行位置、連接認證、執行精靈，並讓代理自行完成初始化設定。

## 引導流程步驟

### 步驟 1：macOS 警告核准

首次啟動時，你必須核准 macOS 的系統安全性警告。

### 步驟 2：本地網路權限

核准尋找本地網路裝置的權限請求。

### 步驟 3：安全性通知

歡迎畫面會顯示安全性資訊。OpenClaw 預設為個人代理：一個受信任的操作者邊界。如果需要共用設定，則需要額外的安全加固措施。

### 步驟 4：Gateway 位置選擇

選擇 Gateway 的執行位置：

- **本機** — 在此 Mac 上本地執行
- **遠端** — 透過 SSH/Tailnet 存取
- **稍後設定** — 延後 Gateway 設定

本機設定允許引導程序在本地設定認證。遠端設定則需要 Gateway 主機上已存在的憑證。

### 步驟 5：權限設定

設定過程會請求以下 TCC 權限：

- 自動化
- 通知
- 輔助使用
- 螢幕錄製
- 麥克風
- 語音辨識
- 攝影機
- 位置

### 步驟 6：CLI 安裝（選用）

應用程式可以透過 npm/pnpm 安裝全域的 `openclaw` 命令列工具，以啟用終端工作流程與 launchd 任務整合。

### 步驟 7：引導對話

設定完成後，會開啟一個專屬的代理對話階段。代理會自我介紹其功能並引導你了解後續步驟。此首次執行導覽與日常對話是分開的。

## 認證注意事項

精靈即使對迴路連線也會產生權杖。本地 WebSocket 用戶端仍需要進行認證。
