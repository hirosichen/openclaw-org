OpenClaw 提供**第一方代理工具**，涵蓋瀏覽器、畫布、節點及排程功能。這些工具取代了舊版的 `openclaw-*` 技能：工具具有型別定義、無需 shell 指令，代理應直接依賴這些工具。

## 停用工具

你可以透過 `openclaw.json` 中的 `tools.allow` / `tools.deny` 全域允許或拒絕工具（拒絕優先）。這會防止被拒絕的工具傳送給模型供應商。

```json5
{
  tools: { deny: ["browser"] },
}
```

注意事項：

* 匹配不區分大小寫。
* 支援 `*` 萬用字元（`"*"` 代表所有工具）。
* 若 `tools.allow` 僅引用未知或未載入的外掛工具名稱，OpenClaw 會記錄警告並忽略允許清單，以確保核心工具保持可用。

## 工具配置檔（基礎允許清單）

`tools.profile` 設定 `tools.allow`/`tools.deny` 之前的**基礎工具允許清單**。可透過 `agents.list[].tools.profile` 進行個別代理覆寫。

配置檔：

* `minimal`：僅 `session_status`
* `coding`：`group:fs`、`group:runtime`、`group:sessions`、`group:memory`、`image`
* `messaging`：`group:messaging`、`sessions_list`、`sessions_history`、`sessions_send`、`session_status`
* `full`：無限制（等同未設定）

範例（預設僅訊息功能，同時允許 Slack + Discord 工具）：

```json5
{
  tools: {
    profile: "messaging",
    allow: ["slack", "discord"],
  },
}
```

範例（coding 配置檔，但全域拒絕 exec/process）：

```json5
{
  tools: {
    profile: "coding",
    deny: ["group:runtime"],
  },
}
```

範例（全域 coding 配置檔，支援代理僅限訊息功能）：

```json5
{
  tools: { profile: "coding" },
  agents: {
    list: [
      {
        id: "support",
        tools: { profile: "messaging", allow: ["slack"] },
      },
    ],
  },
}
```

## 供應商專屬工具政策

使用 `tools.byProvider` 針對特定供應商（或單一 `provider/model`）**進一步限制**工具，而不更改全域預設值。可透過 `agents.list[].tools.byProvider` 進行個別代理覆寫。

此設定在基礎工具配置檔**之後**、允許/拒絕清單**之前**套用，因此只能縮小工具集。供應商鍵接受 `provider`（例如 `google-antigravity`）或 `provider/model`（例如 `openai/gpt-5.2`）。

範例（保持全域 coding 配置檔，但 Google Antigravity 使用最小工具集）：

```json5
{
  tools: {
    profile: "coding",
    byProvider: {
      "google-antigravity": { profile: "minimal" },
    },
  },
}
```

範例（針對不穩定端點的供應商/模型專屬允許清單）：

```json5
{
  tools: {
    allow: ["group:fs", "group:runtime", "sessions_list"],
    byProvider: {
      "openai/gpt-5.2": { allow: ["group:fs", "sessions_list"] },
    },
  },
}
```

範例（針對單一供應商的代理專屬覆寫）：

```json5
{
  agents: {
    list: [
      {
        id: "support",
        tools: {
          byProvider: {
            "google-antigravity": { allow: ["message", "sessions_list"] },
          },
        },
      },
    ],
  },
}
```

## 工具群組（簡寫）

工具政策（全域、代理、沙箱）支援 `group:*` 條目，可展開為多個工具。在 `tools.allow` / `tools.deny` 中使用。

可用群組：

* `group:runtime`：`exec`、`bash`、`process`
* `group:fs`：`read`、`write`、`edit`、`apply_patch`
* `group:sessions`：`sessions_list`、`sessions_history`、`sessions_send`、`sessions_spawn`、`session_status`
* `group:memory`：`memory_search`、`memory_get`
* `group:web`：`web_search`、`web_fetch`
* `group:ui`：`browser`、`canvas`
* `group:automation`：`cron`、`gateway`
* `group:messaging`：`message`
* `group:nodes`：`nodes`
* `group:openclaw`：所有內建 OpenClaw 工具（不包含供應商外掛）

範例（僅允許檔案工具 + 瀏覽器）：

```json5
{
  tools: {
    allow: ["group:fs", "browser"],
  },
}
```

## 外掛 + 工具

外掛可以在核心工具集之外註冊**額外工具**（和 CLI 命令）。參閱 [外掛](/docs/tools/plugin) 了解安裝與設定，以及 [技能](/docs/tools/skills) 了解工具使用指引如何注入提示詞。部分外掛會隨工具一起提供自己的技能（例如語音通話外掛）。

選用外掛工具：

* [Lobster](/docs/tools/lobster)：具備可恢復核准的型別化工作流程執行環境（需要在閘道主機上安裝 Lobster CLI）。
* [LLM Task](/docs/tools/llm-task)：用於結構化工作流程輸出的 JSON-only LLM 步驟（可選 schema 驗證）。

## 工具清單

### `apply_patch`

跨一個或多個檔案套用結構化補丁。適用於多區塊編輯。實驗性功能：透過 `tools.exec.applyPatch.enabled` 啟用（僅限 OpenAI 模型）。`tools.exec.applyPatch.workspaceOnly` 預設為 `true`（僅限工作區）。僅在你有意讓 `apply_patch` 在工作區目錄外寫入/刪除時才設為 `false`。

### `exec`

在工作區中執行 shell 命令。

核心參數：

* `command`（必填）
* `yieldMs`（超時後自動背景執行，預設 10000）
* `background`（立即背景執行）
* `timeout`（秒數；超時後終止處理程序，預設 1800）
* `elevated`（布林值；若已啟用/允許提升模式，則在主機上執行；僅在代理處於沙箱時改變行為）
* `host`（`sandbox | gateway | node`）
* `security`（`deny | allowlist | full`）
* `ask`（`off | on-miss | always`）
* `node`（`host=node` 時的節點 id/名稱）
* 需要真正的 TTY？設定 `pty: true`。

注意事項：

* 背景執行時回傳 `status: "running"` 與 `sessionId`。
* 使用 `process` 來輪詢/檢視記錄/寫入/終止/清除背景工作階段。
* 若 `process` 被禁止，`exec` 將同步執行並忽略 `yieldMs`/`background`。
* `elevated` 受 `tools.elevated` 及 `agents.list[].tools.elevated` 覆寫控制（兩者都必須允許），且為 `host=gateway` + `security=full` 的別名。
* `elevated` 僅在代理處於沙箱時改變行為（否則為無操作）。
* `host=node` 可指向 macOS 配套應用程式或無頭節點主機（`openclaw node run`）。
* 閘道/節點核准與允許清單：[Exec 核准](/docs/tools/exec-approvals)。

### `process`

管理背景 exec 工作階段。

核心操作：

* `list`、`poll`、`log`、`write`、`kill`、`clear`、`remove`

注意事項：

* `poll` 回傳新輸出及完成時的退出狀態。
* `log` 支援基於行的 `offset`/`limit`（省略 `offset` 可取得最後 N 行）。
* `process` 的作用範圍為每個代理；其他代理的工作階段不可見。

### `loop-detection`（工具呼叫循環防護）

OpenClaw 追蹤近期的工具呼叫歷史，在偵測到重複無進展的循環時進行封鎖或警告。透過 `tools.loopDetection.enabled: true` 啟用（預設為 `false`）。

```json5
{
  tools: {
    loopDetection: {
      enabled: true,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      historySize: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

* `genericRepeat`：重複相同工具 + 相同參數的呼叫模式。
* `knownPollNoProgress`：重複輪詢類工具且輸出相同。
* `pingPong`：交替 `A/B/A/B` 無進展模式。
* 個別代理覆寫：`agents.list[].tools.loopDetection`。

### `web_search`

使用 Brave Search API 搜尋網路。

核心參數：

* `query`（必填）
* `count`（1-10；預設來自 `tools.web.search.maxResults`）

注意事項：

* 需要 Brave API 金鑰（建議：`openclaw configure --section web`，或設定 `BRAVE_API_KEY`）。
* 透過 `tools.web.search.enabled` 啟用。
* 回應會被快取（預設 15 分鐘）。
* 參閱 [Web 工具](/docs/tools/web) 了解設定方式。

### `web_fetch`

從 URL 擷取並提取可讀內容（HTML 轉換為 markdown/text）。

核心參數：

* `url`（必填）
* `extractMode`（`markdown` | `text`）
* `maxChars`（截斷過長頁面）

注意事項：

* 透過 `tools.web.fetch.enabled` 啟用。
* `maxChars` 受 `tools.web.fetch.maxCharsCap`（預設 50000）限制。
* 回應會被快取（預設 15 分鐘）。
* 對於 JS 密集型網站，建議使用瀏覽器工具。
* 參閱 [Web 工具](/docs/tools/web) 了解設定方式。
* 參閱 [Firecrawl](/docs/tools/firecrawl) 了解選用的反機器人備援方案。

### `browser`

控制專用的 OpenClaw 管理瀏覽器。

核心操作：

* `status`、`start`、`stop`、`tabs`、`open`、`focus`、`close`
* `snapshot`（aria/ai）
* `screenshot`（回傳圖片區塊 + `MEDIA:<path>`）
* `act`（UI 操作：click/type/press/hover/drag/select/fill/resize/wait/evaluate）
* `navigate`、`console`、`pdf`、`upload`、`dialog`

配置檔管理：

* `profiles` -- 列出所有瀏覽器配置檔及狀態
* `create-profile` -- 建立新配置檔並自動分配連接埠（或 `cdpUrl`）
* `delete-profile` -- 停止瀏覽器、刪除使用者資料、從設定中移除（僅限本地）
* `reset-profile` -- 在配置檔的連接埠上終止孤立程序（僅限本地）

常用參數：

* `profile`（選用；預設為 `browser.defaultProfile`）
* `target`（`sandbox` | `host` | `node`）
* `node`（選用；指定特定節點 id/名稱）

注意事項：

* 需要 `browser.enabled=true`（預設為 `true`；設為 `false` 以停用）。
* 所有操作接受選用的 `profile` 參數以支援多實例。
* 省略 `profile` 時使用 `browser.defaultProfile`（預設為 "chrome"）。
* 配置檔名稱：僅限小寫英數字元 + 連字號（最多 64 字元）。
* 連接埠範圍：18800-18899（最多約 100 個配置檔）。
* 遠端配置檔僅限附加模式（無法啟動/停止/重設）。
* 若有具備瀏覽器功能的節點已連線，工具可能會自動路由至該節點（除非你鎖定 `target`）。
* `snapshot` 在安裝 Playwright 時預設為 `ai`；使用 `aria` 取得無障礙樹狀結構。
* `snapshot` 也支援角色快照選項（`interactive`、`compact`、`depth`、`selector`），回傳如 `e12` 的參考。
* `act` 需要來自 `snapshot` 的 `ref`（AI 快照的數字 `12`，或角色快照的 `e12`）；僅在罕見情況下使用 `evaluate` 進行 CSS 選擇器需求。
* 預設避免 `act` -> `wait`；僅在例外情況下使用（無可靠的 UI 狀態可等待）。
* `upload` 可選擇性傳入 `ref` 以在啟動後自動點擊。
* `upload` 也支援 `inputRef`（aria 參考）或 `element`（CSS 選擇器）直接設定 `<input type="file">`。

### `canvas`

驅動節點 Canvas（呈現、評估、快照、A2UI）。

核心操作：

* `present`、`hide`、`navigate`、`eval`
* `snapshot`（回傳圖片區塊 + `MEDIA:<path>`）
* `a2ui_push`、`a2ui_reset`

注意事項：

* 底層使用閘道 `node.invoke`。
* 若未提供 `node`，工具會選擇預設值（單一已連線節點或本地 mac 節點）。
* A2UI 為 v0.8 版本（無 `createSurface`）；CLI 會拒絕含有行錯誤的 v0.9 JSONL。
* 快速測試：`openclaw nodes canvas a2ui push --node <id> --text "Hello from A2UI"`。

### `nodes`

探索並指定已配對的節點；傳送通知；擷取相機/螢幕畫面。

核心操作：

* `status`、`describe`
* `pending`、`approve`、`reject`（配對）
* `notify`（macOS `system.notify`）
* `run`（macOS `system.run`）
* `camera_list`、`camera_snap`、`camera_clip`、`screen_record`
* `location_get`、`notifications_list`、`notifications_action`
* `device_status`、`device_info`、`device_permissions`、`device_health`

注意事項：

* 相機/螢幕命令需要節點應用程式在前景執行。
* 圖片回傳圖片區塊 + `MEDIA:<path>`。
* 影片回傳 `FILE:<path>`（mp4）。
* 位置回傳 JSON 內容（lat/lon/accuracy/timestamp）。
* `run` 參數：`command` argv 陣列；選用 `cwd`、`env`（`KEY=VAL`）、`commandTimeoutMs`、`invokeTimeoutMs`、`needsScreenRecording`。

範例（`run`）：

```json
{
  "action": "run",
  "node": "office-mac",
  "command": ["echo", "Hello"],
  "env": ["FOO=bar"],
  "commandTimeoutMs": 12000,
  "invokeTimeoutMs": 45000,
  "needsScreenRecording": false
}
```

### `image`

使用設定的圖片模型分析圖片。

核心參數：

* `image`（必填，路徑或 URL）
* `prompt`（選用；預設為 "Describe the image."）
* `model`（選用覆寫）
* `maxBytesMb`（選用大小上限）

注意事項：

* 僅在設定 `agents.defaults.imageModel`（主要或備援）時可用，或當可從預設模型 + 已設定的驗證推斷隱含圖片模型時（盡力配對）。
* 直接使用圖片模型（獨立於主聊天模型）。

### `message`

跨 Discord/Google Chat/Slack/Telegram/WhatsApp/Signal/iMessage/MS Teams 傳送訊息和頻道操作。

核心操作：

* `send`（文字 + 選用媒體；MS Teams 也支援 `card` 用於 Adaptive Cards）
* `poll`（WhatsApp/Discord/MS Teams 投票）
* `react` / `reactions` / `read` / `edit` / `delete`
* `pin` / `unpin` / `list-pins`
* `permissions`
* `thread-create` / `thread-list` / `thread-reply`
* `search`
* `sticker`
* `member-info` / `role-info`
* `emoji-list` / `emoji-upload` / `sticker-upload`
* `role-add` / `role-remove`
* `channel-info` / `channel-list`
* `voice-status`
* `event-list` / `event-create`
* `timeout` / `kick` / `ban`

注意事項：

* `send` 透過閘道路由 WhatsApp；其他頻道直接傳送。
* `poll` 針對 WhatsApp 和 MS Teams 使用閘道；Discord 投票直接傳送。
* 當訊息工具呼叫綁定到活動聊天工作階段時，傳送受限於該工作階段的目標，以避免跨上下文洩漏。

### `cron`

管理閘道排程任務和喚醒。

核心操作：

* `status`、`list`
* `add`、`update`、`remove`、`run`、`runs`
* `wake`（排入系統事件 + 選用的即時心跳）

注意事項：

* `add` 預期完整的 cron 任務物件（與 `cron.add` RPC 相同的 schema）。
* `update` 使用 `{ jobId, patch }`（`id` 為相容性而接受）。

### `gateway`

重新啟動或套用更新到執行中的閘道程序（原地更新）。

核心操作：

* `restart`（授權 + 傳送 `SIGUSR1` 進行程序內重啟；`openclaw gateway` 原地重啟）
* `config.get` / `config.schema`
* `config.apply`（驗證 + 寫入設定 + 重啟 + 喚醒）
* `config.patch`（合併部分更新 + 重啟 + 喚醒）
* `update.run`（執行更新 + 重啟 + 喚醒）

注意事項：

* 使用 `delayMs`（預設 2000）以避免中斷進行中的回覆。
* `restart` 預設啟用；設定 `commands.restart: false` 以停用。

### `sessions_list` / `sessions_history` / `sessions_send` / `sessions_spawn` / `session_status`

列出工作階段、檢視對話記錄，或傳送至另一個工作階段。

核心參數：

* `sessions_list`：`kinds?`、`limit?`、`activeMinutes?`、`messageLimit?`（0 = 無）
* `sessions_history`：`sessionKey`（或 `sessionId`）、`limit?`、`includeTools?`
* `sessions_send`：`sessionKey`（或 `sessionId`）、`message`、`timeoutSeconds?`（0 = 發送後不等待）
* `sessions_spawn`：`task`、`label?`、`runtime?`、`agentId?`、`model?`、`thinking?`、`cwd?`、`runTimeoutSeconds?`、`thread?`、`mode?`、`cleanup?`
* `session_status`：`sessionKey?`（預設為目前；接受 `sessionId`）、`model?`（`default` 清除覆寫）

注意事項：

* `main` 是標準直接聊天鍵；global/unknown 為隱藏。
* `messageLimit > 0` 擷取每個工作階段的最後 N 則訊息（工具訊息已過濾）。
* 工作階段定位由 `tools.sessions.visibility` 控制（預設 `tree`：目前工作階段 + 衍生的子代理工作階段）。若你為多使用者執行共享代理，考慮設定 `tools.sessions.visibility: "self"` 以防止跨工作階段瀏覽。
* `sessions_send` 在 `timeoutSeconds > 0` 時等待最終完成。
* 傳遞/通告在完成後發生且為盡力而為；`status: "ok"` 確認代理執行已完成，而非通告已送達。
* `sessions_spawn` 支援 `runtime: "subagent" | "acp"`（預設 `subagent`）。ACP 執行時行為請參閱 [ACP Agents](/docs/tools/acp-agents)。
* `sessions_spawn` 啟動子代理執行並回覆通告訊息到請求者聊天。
* `sessions_spawn` 為非阻塞式，立即回傳 `status: "accepted"`。

### `agents_list`

列出目前工作階段可透過 `sessions_spawn` 指定的代理 ID。

注意事項：

* 結果受每個代理的允許清單限制（`agents.list[].subagents.allowAgents`）。
* 設定 `["*"]` 時，工具會包含所有已設定的代理並標記 `allowAny: true`。

## 參數（通用）

閘道支援的工具（`canvas`、`nodes`、`cron`）：

* `gatewayUrl`（預設 `ws://127.0.0.1:18789`）
* `gatewayToken`（若啟用驗證）
* `timeoutMs`

注意：設定 `gatewayUrl` 時，請明確包含 `gatewayToken`。工具不會繼承覆寫的設定或環境憑證，缺少明確憑證會導致錯誤。

瀏覽器工具：

* `profile`（選用；預設為 `browser.defaultProfile`）
* `target`（`sandbox` | `host` | `node`）
* `node`（選用；鎖定特定節點 id/名稱）

## 建議的代理流程

瀏覽器自動化：

1. `browser` -> `status` / `start`
2. `snapshot`（ai 或 aria）
3. `act`（click/type/press）
4. `screenshot`（若需要視覺確認）

Canvas 渲染：

1. `canvas` -> `present`
2. `a2ui_push`（選用）
3. `snapshot`

節點定位：

1. `nodes` -> `status`
2. 對選定節點執行 `describe`
3. `notify` / `run` / `camera_snap` / `screen_record`

## 安全性

* 避免直接使用 `system.run`；僅在明確取得使用者同意後使用 `nodes` -> `run`。
* 尊重使用者對相機/螢幕擷取的同意。
* 在呼叫媒體命令前使用 `status/describe` 確認權限。

## 工具如何呈現給代理

工具透過兩個平行通道公開：

1. **系統提示文字**：可供人類閱讀的清單 + 指引。
2. **工具 Schema**：傳送給模型 API 的結構化函式定義。

這意味著代理同時看到「有哪些工具」和「如何呼叫它們」。若工具未出現在系統提示或 schema 中，模型便無法呼叫它。
